C#接口 - Interface

interface与abstract对比

interface功能与abstract类似，都是为多态性考虑而设计的，两者区别在于：

  • interface不能写入字段，不能有具体实现，abstract则相反。
  • interface支持多继承，abstract单继承。
  • interface内所有方法和属性在子类(实现该interface的类)中都得有具体实现，abstract则标有abstract关键字的才需要在子类中有具体实现。

从上述几个角度可以看出，interface可为多继承，限制方法签名，以行为为主体的上下文中使用。
而abstract则可在单继承，类为主体的上下文中使用，可以说各有各用途。同时，这两者也可结合使用，比如在基类派生的子类需要多继承时，则可继承抽象类，并用interface扩充子类。

interface的显示和隐式实现

隐式实现是比较常用的一种方法，即在实现该接口的类中，以public override void xxxxx()的方式实现接口中的方法。

而显示实现，则用于在使用多个接口的类中，多个接口中方法签名重复时，用于表明各个接口方法时使用。形似:public void IFunc.xxxxx()。其中”IFunc”为接口名。

值类型

简介

c#中数据类型分为值类型和引用类型，引用类型很简单就是平时经常用的类，值类型在c#中有一些默认的值类型如：int,float,double,char等。(string是引用类型)
当然object为所有值类型和引用类型的基类。C# 终极基类Object介绍
而值类型也称为结构struct，与类class对应。（这里的struct与c++中不同）

一般来说因为结构不能传递引用，复制值类型数据时都是在栈上直接复制数据的，消耗比引用类型的地址大，(值类型不存在gc问题),且存在装箱拆箱问题，所以除了预定义的一些值类型外不会经常使用。
